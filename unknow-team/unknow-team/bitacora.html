Bitácora del Proyecto

Documentación del proceso de desarrollo, decisiones de diseño y dificultades encontradas durante la creación del sitio web de UNKNOW.

📅 17 de Abril — Inicio del Proyecto

Jueves, 17 de Abril de 2025

Reunión inicial del equipo para definir los objetivos fundamentales del proyecto UNKNOW. Decidimos orientar el diseño hacia una estética editorial inspirada en revistas de moda combinada con tecnología web contemporánea.

  • Reunión inicial del equipo — Presentación de todos los integrantes y definición de roles.
  • Definición de objetivos y alcance — Establecimos que la web debía ser 100% responsive con al menos tres breakpoints.
  • Asignación de roles y responsabilidades — Carlos (backend/estructura), Heber (frontend/CSS), Javier (JS/interactividad), Leandro (diseño/UX).
  • Decisión de diseño — Elegimos Bento Grid como layout principal por su versatilidad responsiva.

🎨 18 de Abril — Diseño UI/UX

Viernes, 18 de Abril de 2025

Sesión de diseño creativo enfocada en la identidad visual del equipo. La mayor dificultad fue ponernos de acuerdo en la paleta de colores; terminamos resolviendo con una votación interna.

  • Creación de moodboards — Buscamos referencias en Dribbble, Awwwards y revistas como i-D y Vogue.
  • Diseño de wireframes en Figma — Bocetamos la estructura de la portada y las tarjetas individuales.
  • Selección de paleta de colores — Definimos #e6e3dc (fondo beige), #ff2a2a (rojo), #0d0c0c (texto).
  • Tipografías — Playfair Display para títulos (elegancia serif), Google Sans para cuerpo (legibilidad).
  • Dificultad — Varios integrantes preferían un fondo oscuro; lo resolvimos implementando el efecto Morfosis como modo oscuro togglable.

💻 21 de Abril — Estructura HTML

Lunes, 21 de Abril de 2025

Implementación de la estructura semántica base del sitio. Definimos la organización de archivos aplicando buenas prácticas: HTML en raíz, CSS en /css, JS en /js, imágenes en /img, páginas individuales en /Integrantes.

  • HTML semántico — Uso de header, nav, section, footer y atributos ARIA para accesibilidad.
  • Organización de archivos — Estructuramos el repositorio según los requerimientos del TP.
  • Template de tarjetas — Creamos un template base para las páginas de cada integrante.
  • Dificultad — La navegación entre páginas con View Transitions API requería que los links usaran view-transition-name; tuvimos que reorganizar varios IDs que colisionaban.

🎨 22 de Abril — CSS y Responsive Design

Martes, 22 de Abril de 2025

Jornada intensa de CSS. Implementamos el sistema de Bento Grid para la portada y las tarjetas individuales, asegurándonos de que todo funcionara correctamente en los tres breakpoints requeridos.

  • Bento Grid — Implementación con CSS Grid usando grid-template-columns y grid-column: span.
  • Breakpoints — Media queries para ≤400px (mobile), ≤900px (tablet), ≤1200px (desktop grande).
  • Glassmorphism del header — backdrop-filter + position: sticky para la navegación pegajosa.
  • Dificultad — El Bento Grid colapsaba mal en móviles pequeños. Lo solucionamos usando !important solo en el breakpoint de 400px para forzar grid-column: span 1 en todas las cards.

⚡ 24 de Abril — Funcionalidades JavaScript

Jueves, 24 de Abril de 2025

Desarrollo de interactividad y efectos visuales. Implementamos las tres funciones JS principales: efecto fuego, reproductor de música y efecto morfosis, con persistencia usando sessionStorage.

  • Efecto de Fuego Toggle — Aplica gradiente animado sobre títulos h1/h2 con CSS @keyframes. Activable desde el botón del navbar.
  • Reproductor de Música — Play/Pause real usando la API Audio de JavaScript con control de estado entre canciones.
  • Efecto Morfosis — Cambia el tema de la página a modo oscuro usando classList.toggle y persiste con sessionStorage.
  • Dificultad — La View Transitions API no está disponible en todos los navegadores. Implementamos un polyfill suave que navega normalmente si no está disponible.
  • Cambio importante — Decidimos almacenar el estado del fuego y morfosis en sessionStorage para que persistan al navegar entre páginas sin usar localStorage global.

📱 25 de Abril — Testing y Ajustes

Viernes, 25 de Abril de 2025

Jornada de testing en múltiples dispositivos y navegadores. Encontramos y corregimos varios problemas de layout y comportamiento de los scripts.

  • Testing en Firefox — View Transitions no disponible, confirmamos que el fallback funciona correctamente.
  • Testing en móvil Android — Las cards del bento grid se superponían en 360px. Corregimos con un media query adicional.
  • Corrección del reproductor — El audio seguía sonando al navegar a otra página. Solucionamos pausándolo en el evento pagehide.
  • Dificultad — backdrop-filter no funcionaba en algunos navegadores sin prefijo -webkit-. Agregamos ambas versiones.

🚀 28 de Abril — Deploy y Documentación

Lunes, 28 de Abril de 2025

Etapa final: subida del repositorio a GitHub, deploy en Vercel y redacción del README.md completo con todas las secciones requeridas.

  • Repositorio GitHub — Creación del repo, .gitignore y primeros commits por integrante.
  • Deploy en Vercel — Configuración del proyecto y verificación del dominio: unknow-team.vercel.app.
  • README.md — Documentación completa con paleta de colores, tipografías, funciones JS y uso de IA.
  • Cambio importante — Eliminamos el hero redundante de las páginas de integrantes y migramos todo a un Bento Grid puro, lo que mejoró significativamente la legibilidad en móviles.

🔁 30 de Abril — Revisión Final

Miércoles, 30 de Abril de 2025

Revisión general del proyecto antes de la entrega. Verificamos que todos los requerimientos obligatorios estuvieran cubiertos y mejoramos detalles de diseño menores.

  • Checklist de requerimientos — Confirmamos: portada ✅, tarjetas individuales ✅, bitácora ✅, responsive ✅, JS interactivo ✅, README ✅, uso de IA ✅.
  • Mejoras de detalle — Ajustamos el espaciado del header en mobile, refinamos las animaciones del efecto fuego.
  • Reflexión del equipo — El mayor aprendizaje fue el trabajo con CSS Grid avanzado y la coordinación entre branches de Git sin conflictos de merge.
